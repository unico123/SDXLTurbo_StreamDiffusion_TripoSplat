# CCPM + Ralph 工作流选择指南

> **选择适合你的项目工作流** - 两种模式，各有优势

---

## 🎯 快速选择

### 你需要哪种工作流？

**回答以下问题，找到最适合的工作流：**

1. **你的项目类型？**
   - 个人项目 → 🏠 本地优先工作流
   - 团队项目 → 🌐 GitHub集成工作流

2. **你的项目大小？**
   - 大型项目 (10GB+) → 🏠 本地优先工作流
   - 中小型项目 → 两种都可以

3. **你的协作需求？**
   - 需要多人协作 → 🌐 GitHub集成工作流
   - 独立开发 → 🏠 本地优先工作流

4. **你的网络环境？**
   - 经常离线 → 🏠 本地优先工作流
   - 稳定在线 → 两种都可以

---

## 🌐 GitHub集成工作流

**适用场景**：
- ✅ 团队协作项目 (2人以上)
- ✅ 开源项目
- ✅ 需要远程访问
- ✅ 需要代码审查
- ✅ 分布式团队

**核心优势**：
- 🤝 **团队协作** - GitHub Issues 作为协作中心
- 🔍 **代码审查** - PR 流程和代码审查
- 🌍 **远程访问** - 随地访问和协作
- 📊 **进度透明** - 团队成员实时看到进度

**详细指南**：[GITHUB_INTEGRATION_WORKFLOW.md](GITHUB_INTEGRATION_WORKFLOW.md)

**典型命令**：
```bash
"创建功能X"         # CCPM规划
"同步到GitHub"      # CCPM同步
ralph --github-issue 123 --monitor  # Ralph执行
```

---

## 🏠 本地优先工作流

**适用场景**：
- ✅ 个人项目
- ✅ 大型工程 (50GB+)
- ✅ 敏感/私有项目
- ✅ 离线开发环境
- ✅ 快速原型开发

**核心优势**：
- ⚡ **极致性能** - 无网络延迟，即时响应
- 💾 **无大小限制** - 处理超大项目无压力
- 🔒 **完全控制** - 代码和数据完全私有
- 🚀 **离线工作** - 完全不依赖网络

**详细指南**：[LOCAL_FIRST_WORKFLOW.md](LOCAL_FIRST_WORKFLOW.md)

**典型命令**：
```bash
"创建功能X"         # CCPM本地规划
ralph --monitor      # Ralph本地执行
```

## 🎯 推荐决策

```
需要团队协作？ → GitHub集成
大型项目(10GB+)？ → 本地优先
需要远程访问？   → GitHub集成
经常离线？       → 本地优先

默认推荐：本地优先工作流 (简单高效)
```
